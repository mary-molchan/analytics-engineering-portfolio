# ❄️ Projets Snowflake SQL

Ce dossier regroupe plusieurs projets Snowflake SQL réalisés dans une logique d’analytics engineering, avec un focus sur la préparation des données, l’industrialisation des traitements, la création de couches de reporting, la qualité des données et la documentation technique.  Ces projets montrent comment SQL peut être utilisé non seulement pour interroger des données, mais aussi pour construire des pipelines analytiques structurés, alimenter des tables de synthèse, préparer des vues BI consolidées et rendre les traitements plus fiables, maintenables et réutilisables.

## 🧩 Périmètre technique

Les projets présentés dans cette section couvrent différents cas d’usage SQL / Snowflake liés à la préparation et à l’industrialisation des données pour le reporting décisionnel.

Ils peuvent inclure :

- des scripts SQL et des procédures stockées ;
- la logique de préparation des données dans Snowflake ;
- des pipelines de chargement et de transformation ;
- des tables physiques de reporting et de synthèse ;
- des couches RAW, MART ou BI-ready ;
- la mise en œuvre de règles métier ;
- des calculs d’indicateurs financiers et opérationnels ;
- des contrôles qualité et validations de données ;
- des traitements incrémentaux et des fenêtres historiques ;
- des spécifications techniques au format Markdown ;
- une documentation claire de l’ordre d’exécution ;
- une structure de projet réutilisable pour les analyses basées sur SQL.

## 🎯 Objectif de cette section

Cette section du portfolio vise à démontrer ma capacité à structurer des projets SQL / Snowflake proches de cas d’usage réels en entreprise : préparation de données, fiabilisation des traitements, alimentation de reportings décisionnels, documentation technique et mise en place de pipelines maintenables.
